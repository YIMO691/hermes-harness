# docs/

Hermes Harness 工程文档。

## 放什么

- 架构文档
- 工程计划（v1.8 / v1.9 / v1.10）
- 治理裁决
- 评估报告
- 证据报告
- 领域覆盖地图
- 仓库结构治理规则
- 历史归档（`SUPERSEDED_*`）

## 不放什么

- 单次任务产物（WorklineSummary.md、metrics-lite.yaml） → 放 `tasks/`
- 运行时 skill → 放 `skills/`
- checker fixture → 放 `tests/`

## 分组导航

参见 `docs/README.md`（本文件）。

## 规则

- 架构文档命名：`WORKLINE_ARCHITECTURE_V*_STABLE.md`
- 版本计划命名：`V*_*_PLAN.md`
- 评估报告放：`evaluations/<version>/`
- 历史归档加 `SUPERSEDED_` 前缀
